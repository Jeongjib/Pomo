# 1-Pager — 모션캡처 기반 자동 프라이버시 앱 (Pomo, 가칭)

## 한 줄 요약
노트북 카메라로 누군가 다가오는 신호를 감지해 카톡/딴짓 앱을 자동으로 숨겨주는 macOS 네이티브 앱. **Pomodoro 타이머로 위장**해서 평상시에도 자연스럽게 작동.

---

## 문제

사무실에서 PC 카톡, 개인 브라우저 등을 잠깐 쓰다가 상사/동료에게 들킬까 봐 신경 쓰이는 회색지대 상황.

- 명시적으로 금지는 아니지만 사회적으로 어색
- Boss key는 수동이라 늦고, 프라이버시 필름은 시야각 문제만 해결
- macOS Voice Control / 접근성 도구는 학습 비용이 높고 범용 제어용
- 음성 명령은 옆 사람 있을 때 못 씀

→ **조용히 + 손 안 쓰고 + 학습 부담 0** 으로 작동하는 솔루션이 비어있음.

---

## 솔루션

노트북 카메라로 두 가지 **수동적 신호**를 감지해 자동으로 민감 앱을 hide. 사용자가 명령을 외울 필요 없음, 의식할 필요도 없음.

### 트리거 1 — 본인의 본능적 head turn
- 0.3초 이내 yaw 변화 > 40°
- "뒤에 누가 왔다"는 신체 반응을 잡음
- **카메라가 등 뒤를 못 보는 한계를 우회하는 핵심 아이디어**

### 트리거 2 — 타인 얼굴 + 화면 응시
- 프레임 내 2번째 얼굴 검출
- 그 얼굴의 **head orientation** 이 화면 방향을 1초+ 유지
- (gaze estimation 대신 head pose로 단순화 → 거리/조명에 robust)
- 정적 얼굴(벽 사진 등)은 미세 움직임 부재로 필터링

두 트리거가 보완적:
- (1) = 선제적 감지 (사용자가 먼저 알아챔)
- (2) = 사후 감지 (이미 봤지만 그 이상 못 보게)

---

## 트리거 발동 시 동작

1. **민감 앱 hide** — `NSRunningApplication.hide()` for 사전 등록된 bundle ID들
2. **업무 앱 frontmost로 activate** (선택)
3. **Focus mode / DnD 활성화** — 카톡 알림 배너 차단 필수
4. **"쉬는 시간 종료" 알림 띄우기**
   - 카피: "10분 휴식이 끝났어요. 업무로 복귀할 시간이에요"
   - **alibi 구축**: 딴짓 → 스케줄된 휴식으로 프레이밍
5. **수동 복귀** — 자동 unhide 안 함. 사용자가 단축키로 직접 복원
   - 이유: 자동 복귀 false positive 비용이 너무 큼 (상사가 옆에 있는데 복귀되면 끝)

---

## 차별점 — Pomodoro 앱으로 위장

제품 정체성을 통째로 생산성 앱으로 포지셔닝:

- Dock에 보이는 이름: **"Pomo"** 또는 **"Focus"**
- 평상시 기능: 진짜 작동하는 뽀모도로 타이머
- "쉬는 시간 종료" 알림이 **평소에도 자연스럽게 뜸** → 트리거 발동 시 "왜 하필 지금?" 의심 차단
- App Store 심사 통과 가능 (실제 생산성 앱)
- 친구/동료에게 추천해도 어색하지 않음
- 보안팀이 앱을 봐도 의심 안 함

**비밀스러운 hide 기능은 부가 기능, 뽀모도로가 표면. 양쪽 다 진짜 가치를 줌.**

---

## 기술 스택 (모두 native macOS)

| 영역 | 기술 |
|---|---|
| 앱 셸 | Swift + AppKit (메뉴바 앱) |
| 카메라 캡처 | AVFoundation, 5~10 fps |
| 얼굴/포즈 검출 | Vision framework (`VNDetectFaceLandmarksRequest`, head pose) |
| 앱 제어 | `NSRunningApplication.hide()` / `.activate()` |
| 알림 | UserNotifications framework |
| Focus mode 토글 | `shortcuts run` CLI 또는 Shortcuts URL scheme |

외부 의존성 거의 없음. 카메라 권한만 필요 (Accessibility 권한 불필요).

## MVP 범위 (1주 프로토타입)

- [ ] 카메라 캡처 + Vision face detection 파이프라인
- [ ] 트리거 1 (head turn) 만 구현
- [ ] 민감 앱 1개(카톡) 사전 등록 후 hide
- [ ] "쉬는 시간 종료" 네이티브 알림
- [ ] 메뉴바 토글 (on/off, 카메라 상태 표시)
- [ ] **본인 1주 dogfooding** — 실제로 켜놓는지 검증

검증 통과 시 다음 단계:
- 트리거 2 (다른 얼굴 감지) 추가
- Pomodoro 타이머 UI 본격 구현
- 다중 앱 등록, 알림 커스터마이징
- 복귀 단축키 설정

## 알려진 한계 / 엣지 케이스

- **카메라 위치**: 등 뒤 직접 접근은 카메라가 못 봄 → 트리거 1로 우회
- **외장 모니터 / 클램셸 모드**: 외장 카메라 필요. 노트북 단독 사용 시나리오를 우선 타겟
- **저조도**: 어두운 환경에선 얼굴 검출 정확도 저하. 사무실은 대체로 밝아서 큰 문제 아님
- **카메라 상시 켜짐**: 사용자 거부감 가능 → 메뉴바에서 카메라 작동 상태 명확히 표시 (신뢰 신호)
- **배터리/발열**: 저frame rate + 경량 ML이라 부담 작음. 발열 모니터링 필요
- **트리거 1 false positive**: 시계 보기, 동료가 부름 등 → hide 비용이 낮으므로 수용 가능 (cmd+tab으로 복원)
- **윤리/포지셔닝**: "딴짓 도구"로 보이지 않게 Pomodoro 프레임 필수
